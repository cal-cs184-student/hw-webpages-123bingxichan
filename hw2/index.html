Each step finds the intermediate control point by linearly interpolating between two neighboring control points. Rinse and repeat until we reach a single point — and this point lies on the Bezier curve. I implemented the algorithm exactly as described in the spec.

Part 2: Bezier Surfaces with Separable 1D de Casteljau

In Part 1, I implemented de Casteljau's algorithm for 2D Bezier curves. In Part 2, we extend to 3D Bezier surfaces by applying the algorithm twice — once in each direction.

A Bezier surface is defined by an n×n grid of control points. Given parameters (u, v), you first run the full 1D de Casteljau along each row using u, collapsing each row of n points down to a single point. Then you run it one more time on those n points using v to get the final surface point.

In code, evaluateStep is nearly identical to the curve version. evaluate1D repeatedly calls evaluateStep until one point remains. evaluate loops over each row, calling evaluate1D with u to collapse each row, then calls evaluate1D one final time on those results with v.

Part 3: Area-Weighted Vertex Normals

To get the area-weighted vertex normal, I looped over all neighboring faces of the vertex using the half-edge structure. For each face I grabbed two edge vectors going out from the vertex and took their cross product. The cross product gives you both the normal direction and the area at once — its magnitude is proportional to the triangle area, so area-weighting comes for free. I accumulated all those cross products into a sum and normalized at the end.

Part 6: Loop Subdivision for Mesh Upsampling

Using the weight computations defined in the spec, I looped through all edges and vertices and stored the updated positions in their newPosition attributes. I then collected all original edges before splitting into a vector, called split edge on each, and moved the new vertex to the saved point. To flip, I checked whether a new edge connects one old and one new vertex. Finally, I iterate through every vertex and move it to its precomputed new position.

The main observation is that sharp features get smoother with each subdivision — particularly noticeable on the teapot spout.

Cube Subdivision Experiments

When I upsampled a cube repeatedly without any pre-processing, it became a potato shape. However, when I pre-split edges on some faces and left others alone, I noticed the pre-split faces remained more cube-like while the untouched faces rounded out more aggressively. This makes intuitive sense — more edges give the subdivision algorithm more local detail to preserve.
