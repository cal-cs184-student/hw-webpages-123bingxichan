Section I: Bezier Curves and Surfaces

Part 1: Bezier curves with 1D de Casteljau subdivision

To me, de Casteljau's algorithm is a way to calculate the value of a Bezier curve at a certain point.To my understanding, each step of de Casteljau's finds the intermediate control point by finding the midpoint of the 2 control points it is bounded by (if t = 0.5), otherwise it linearly inerpolated to find this midpoint. Rinse and repeat until we reach a 1D point, and this point lies on the bezier curve. I lowkenuinely just implemented the algorithm as described in the spec.

Part 2: Bezier surfaces with separable 1D de Casteljau

In Part 1, I implemented de Casteljau's algorithm for 2D bezier curves, but in Part 2 we have 3D bezier surfaces.De Casteljau extends to surfaces by just applying the algorithm twice, once in each direction. A Bezier surface is defined by an n x n grid of control points. Given parameters (u, v), you first run the full 1D de Casteljau along each row using u, which collapses each row of n points down to a single point, giving you n points total, one per row. Then you run it one more time on those n points using v, and that gives you the final point on the surface. In code, evaluateStep is basically identical to the curve version, but it takes a vector of 3D points and parameter t and returns one fewer intermediate point via lerp. evaluate1D just repeatedly calls evaluateStep until one point remains. evaluate finally loops over each row of controlPoints, calling evaluate1D with u to collapse each row to a single point, collecting those into a vector, and finally calling evaluate1D one more time on that vector with v to get the surface point.

Section II: Triangle Meshes and Half-Edge Data Structure

Part 3: Area-weighted vertex normals

To get the area weighted vertex normal, I just looped over all neighboring faces of the vertex using the halfedge structure. For each face I grabbed two edge vectors going out from the vertex and took their cross product. The cool thing is the cross product gives you both the normal direction and the area at once since its magnitude is proportional to the triangle area, so you get area weighting for free. I accumulated all those cross products into a sum and normalized at the end.
